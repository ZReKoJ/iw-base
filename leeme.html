Robotron 2000

Repositorio github

Descripción

    Robotron 2000 es un juego para el navegador, en el que programas el comportamiento de un robot en Javascript para competir contra robots programados por otros usuarios.

    Define qué acciones debe tomar tu robot para explorar el mapa, buscar a tu enemigo y derrotarlo.

    Diseña, edita y guarda los códigos de tus robots.

    Diseña, edita y guarda tus mapas personalizados.

    Juega y compite contra los robots de otros usuarios en mapas creados por ti o por otros usuarios.

    Gana partidas y escala posiciones en el ranking.

Comentarios al profesor

La distribución del trabajo dentro del equipo la hemos decidido hacer de la siguiente forma. Podemos dividir más o menos el proyecto en tres partes:

  • Presentación: Los cuales son las vistas, y se encarga Lorenzo, cuyo tarea también incluye en escribir y reorganizar los archivos css.
  • Negocio: Se encarga Zihao, haciéndose responsable de conectar lógicamente las distintas partes del proyecto y escribiendo los archivos js.
  • Integración: Parte donde se hace el intercambio de datos entre la aplicación y base de datos. Se responsabiliza César, el cual crea en código el modelo de dominio y hace sus conexiones.

Lo decidimos así para que podamos trabajar en paralelo y así minimizar lo máximo los conflictos, a parte de que si uno se encarga de una capa no se vería diferencias de estilo de código en un mismo fichero. A medida que vamos avanzando, cada uno enseña a los otro dos como está haciendo su parte.

Por otro lado, otro de los retos a los que nos hemos enfrentado es a la evasión de acciones malintencionadas como podría ser la manipulación del resultado de las partidas. Hemos conseguido hacer nuestra aplicación bastante robusta, evitando que un usuario pueda acceder a los datos de otro a través de peticiones al servidor. Con estas medidas evitamos:

  • Edición de mapas ajenos
  • Eliminación de mapas ajenos
  • Edición de códigos ajenos
  • Eliminación de códigos ajenos

No obstante, y debido a que la ejecución del código se ejecuta en el lado cliente, no podemos asegurar que los datos que se reciben en el lado servidor referentes a cada partida sean legítimas.

Vistas

Main Page

http://localhost:8080/

Página principal del juego. Los elementos principales de esta página son los botones de Menu y Play y el nombre y breve descripción del juego. Al hacer click en el botón de menú se mostrarán el resto de secciones de la aplicación a las que se podrá navegar desde aquí:

  • Play
  • Ranking
  • Profile
  • Code Design
  • Map Design

Play

http://localhost:8080/play

En esta sección podemos seleccionar 3 elementos:

  • Select Your Robot: este menú desplegable muestra una lista de códigos previamente creados por el usuario de la sesión actual. De esta lista el usuario deberá elegir el código con el que va a jugar la partida
  • Select Map: este menú desplegable muestra una lista de mapas previamente creados por los usuarios, no necesariamente por el usuario de la sesión actual, y se deberá elegir uno de ellos para jugar.
  • Select Enemy Robots: este menú desplegable muestra la lista de todos los códigos creados por los usuarios. De esta lista el usuario podrá elegir hasta un máximo de 5 códigos enemigos diferentes. Estos códigos pueden pertenecer al mismo usuario, incluso al usuario de la sesión actual.
  • Preview del mapa: podrás ver el mapa seleccionado.
  • Preview del código: podrás ver el código seleccionado.
  • Botón Play: este botón se mantendrá desactivado hasta que el usuario elija todos los elementos de la partida. Una vez seleccionados dichos elementos el botón se activará y redirigirá al usuario a la vista del juego

Playing

http://localhost:8080/playing

En esta vista se llevará a cabo la ejecución de la partida de manera automática. Los robots se comportarán acorde a los códigos seleccionados previamente. Batallarán entre ellos hasta que solo quede 1. Los robots tienen unos valores de ataque y defensa base. Los cálculos de daño se calculan en función de el valor de ataque del atacante y el valor de defensa del atacado. Además se mueven a una velocidad determinada. Estas tres características se ven afectadas por los elementos del terreno sobre los que esté situado el robot.

Durante el juego el usuario solo podrá espectar la partida y hacer zoom (con la rueda del ratón) para ver con más detalle la partida, además de seguir continuamente los movimientos de un robot. Esta acción se realiza clickando en barra de vida del robot que se quiere seguir. El robot seguido se mostrará en con un asterisco al lado del nombre.

Barras de vida: se situa en la parte superior derecha de la vista con los siguientes datos: nombre del código/nombre del creador --- porcentaje de la vida del robot,número de balas

Botón Play again: este botón se mantendrá desactivado hasta el final de la partida. Una vez finalizada, permitirá al usuario volver a la pantalla de configuración de partida (vista play).

Botón Play/Pause: controlar el desarrollo del juego, puedes parar el juego y guardar el instante del juego en un imagen y despues iniciar otra vez el juego.

Botón FullScreen: permite al usuario introducirse en el modo pantalla completa. Para salir de él se pulsará el botón Escape.

Code Design

http://localhost:8080/code-design

En esta vista cada usuario podrá editar, cargar, y guardar sus códigos. Dichos códigos determinarán el comportamiento de los robots en partida. La herramienta de edición de código resaltará de manera inteligente los elementos típicos de JavaScript. Es necesario nombrar el código para poder guardarlo.

Se puede editar código directamente en esta vista o subir código desde tu ordenador.

Map Design

http://localhost:8080/map-design

En esta vista cada usuario podrá crear y guardar sus mapas, los cuales serán utilizados como campos de batalla en las partidas. Para editar tu mapa deberás seleccionar el tipo de casilla que quieres agregar desde el menú de la derecha y hacer click en las casillas donde quieras colocarlas. Puedes navegar por el mapa al igual que en Play con la rueda del ratón para hacer zoom y desplazarte por él. Es necesario nombrar el mapa para guardarlo.

El usuario podrá elegir el número de filas y columnas con la herramienta de dimensionamiento

Los elementos sobre los que el robot puede desplazarse son (nombrados de arriba a abajo y de izquierda a derecha):

  • Borrador: este elemento de color negro permite eliminar un elemento del mapa.
  • Plataforma: el robot es capaz de desplazarse sobre este elemento a velocidad x2. Estando sobre este elemento mantiene su valor de ataque y defensa.
  • Tierra: el robot es capaz de desplazarse sobre este elemento a velocidad x0,8. Estando sobre este elemento mantiene su valor de ataque y su defensa aumenta un 50%.
  • Hierba: el robot es capaz de desplazarse sobre este elemento a velocidad x0,5. Estando sobre este elemento mantiene su valor de ataque y su defensa aumenta un 100%.
  • Puente: el robot es capaz de desplazarse sobre este elemento a velocidad x0,3. Estando sobre este elemento aumenta su valor de ataque un 50% y su defensa aumenta un 150%.
  • Magma: el robot es capaz de desplazarse sobre este elemento a velocidad x0,4. Estando sobre este elemento aumenta su valor de ataque un 150% y su defensa disminuye un 20%.

Profile

http://localhost:8080/profile

Aqui el usuario podrá visualizar y editar su perfil. Se mostrará toda la informacion del usuario: foto de perfil, mensaje de bienvenida junto al nombre, número de victorias y derrotas, chat para comunicarse con el resto de usuarios conectados, lista de códigos creados, lista de mapas creados, botón de logout y botón back para volver a menú principal

Ranking

http://localhost:8080/ranking

En esta vista se muestra el ranking de todos los usuarios junto con las victorias y derrotas de cada uno. El ranking está ordenado por número de victorias. En caso de empate, quedará en mejor posición el usuario con menos derrotas. En caso de haber empate también en este valor, quedará en mejor posición el usuario más veterano
